Awan Saya

The connectivity platform for Tela. Manage machines, services, and encrypted tunnels across all your hubs from a single dashboard.

Sample Portal Download Tela CLI

Why Awan Saya?

One platform. All your hubs. Every machine.

🌐

Multi-Hub Aggregation

See every machine across every hub in one dashboard. No more switching between consoles.

🔒

End-to-End Encrypted

Tela tunnels use WireGuard encryption. The hub never sees your traffic — only metadata.

⚡

Zero Config Networking

STUN hole-punching with automatic relay fallback. Works through NAT, firewalls, and CGN.

💻

Cross-Platform CLI

One binary for Windows, Linux, and macOS. Log in to your portal once, then use short hub names—no config files needed.

🏢

Federation

Any Tela hub can be registered with any portal. Hub owners keep full control of their infrastructure.

👤

User-Centric Access

Users see machines, not infrastructure. Grant access without exposing hub topology.

How It Works

  1. Deploy a hub (relay) and register it Download telahubd and run it on a machine reachable by your agents and clients—a VPS, cloud VM, or an edge box on your network. A single command starts the hub:
    telahubd
    By default it listens on port 8080 (HTTP+WS) and 41820 (UDP relay). Set HUB_NAME for the portal display name. Then add it to the Awan Saya portal with a short name and URL.
  2. Install the agent on each machine Download telad and run it on every machine you want to access remotely. It connects to your hub and advertises the machine’s services—SSH, RDP, HTTP, or any TCP port.
  3. Connect from anywhere On your workstation, log in to your portal once:
    tela login https://awansaya.net
    Then connect using short hub and machine names:
    tela connect -hub <hub-name> -machine <machine-name>
    The CLI resolves hub names via the portal, establishes an encrypted WireGuard tunnel, then automatically upgrades to a direct peer-to-peer connection when possible.
  4. Manage from the portal Open the Awan Saya portal to see all your hubs, machines, and active sessions in one place. Grant access to teammates without sharing credentials or exposing infrastructure.
